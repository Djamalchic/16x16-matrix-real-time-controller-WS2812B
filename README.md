# WS2812B 16x16 LED Matrix Controller 🎨🤖

Инструмент для управления светодиодной матрицей 16x16 (WS2812B) в реальном времени с помощью ПК через ESP32 / ESP32-S3. Проект состоит из прошивки для ESP32 и десктопного графического интерфейса на Python (Tkinter).

---

## 🌟 Основные возможности

* **Визуальный редактор сетки**: Интуитивное рисование на матрице 16x16 прямо с компьютера.
* **Синхронизация в реальном времени**: Мгновенная передача пикселей на ESP32 через высокоскоростное последовательное соединение (Serial, 115200 baud).
* **Удобный выбор цвета**: Встроенная палитра быстрых цветов + стандартный диалог выбора цвета (Color Chooser).
* **Два режима рисования**:
  * Левая кнопка мыши (ЛКМ) — рисование выбранным цветом.
  * Правая кнопка мыши (ПКМ) — ластик (черный цвет).
* **Поддержка Zigzag-разметки (Змейка)**: Переключение между прямой и зигзагообразной адресацией светодиодов.
* **Настройка яркости**: Плавный слайдер регулировки яркости (от 1 до 255).
* **Групповая отправка пикселей (Batching)**: Для оптимизации скорости и предотвращения задержек при заливках или перерисовках.
* **Экспорт в Arduino код**: Сохранение созданного на сетке рисунка в готовый `.ino` файл в виде массива статических команд `strip.setPixelColor(...)` для автономной работы матрицы.

## 📁 Структура репозитория

* 📄 `matrix_controller.py` — Десктопное приложение на Python (Tkinter + PySerial) для управления матрицей.
* 📄 `esp32_firmware.ino` — Прошивка для платы ESP32 / ESP32-S3 (Arduino Sketch).
* 📄 `requirements.txt` — Список зависимостей Python.

> Дополнительные папки `facefusion` и `deep` не требуются для работы данного контроллера и могут быть удалены или перенесены в другие репозитории.

## 🛠 Требования и подготовка к работе

### 1. Аппаратная часть (Hardware)
* Плата управления: **ESP32** / **ESP32-S3** (или любая совместимая Arduino-плата с USB-Serial интерфейсом).
* Светодиодная матрица **WS2812B** размером **16x16** (256 пикселей).
* Подключение:
  * **VCC** матрицы ➡️ **5V** питания (при большой яркости рекомендуется использовать внешний источник питания 5V, объединив GND).
  * **GND** матрицы ➡️ **GND** платы ESP32.
  * **DIN (Data In)** матрицы ➡️ **GPIO 48** (настраивается в прошивке).

---

### 2. Установка прошивки на ESP32

1. Установите среду разработки **Arduino IDE**.
2. Установите библиотеку **Adafruit NeoPixel**:
   * Перейдите в *Скетч* ➡️ *Подключить библиотеку* ➡️ *Управлять библиотеками...*
   * Найдите и установите **Adafruit NeoPixel**.
3. Откройте файл `esp32_firmware.ino`.
4. При необходимости измените пин подключения матрицы в строке:
   ```cpp
   #define LED_PIN 48 // Измените на пин вашей платы
   ```
5. Выберите вашу плату ESP32 и соответствующий COM-порт в настройках Arduino IDE.
6. Загрузите (Flash) скетч в плату. После успешной прошивки в мониторе порта появится надпись `READY`.

---

### 3. Запуск Python-клиента

1. Убедитесь, что у вас установлен Python версии 3.x.
2. Клонируйте репозиторий и перейдите в его папку.
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустите управляющее приложение:
   ```bash
   python matrix_controller.py
   ```

## 🎮 Инструкция по использованию

1. **Подключение**:
   * Подсоедините ESP32 к компьютеру через USB.
   * Запустите Python-клиент.
   * В блоке **«Подключение»** выберите нужный COM-порт (если порты не отображаются, нажмите кнопку обновления 🔄).
   * Выберите режим разметки вашей матрицы (**«Змейка (zigzag)»**, если диоды идут змейкой, или оставьте выключенным для стандартного построчного подключения).
   * Нажмите кнопку **«Подключить»**. Статус изменится на зеленый маркер с именем порта.
2. **Рисование**:
   * Кликните на цветной прямоугольник или выберите пресет из палитры, чтобы задать цвет.
   * Нажмите левую кнопку мыши на сетке и ведите курсор, чтобы закрасить пиксели.
   * Используйте правую кнопку мыши, чтобы стирать пиксели (устанавливать черный цвет).
3. **Действия**:
   * **Очистить матрицу**: Полностью выключает все светодиоды (черный цвет).
   * **Залить цветом**: Закрашивает всю матрицу выбранным цветом.
   * **Отправить всё на ESP**: Принудительно передает текущую картинку с экрана в память ESP32 пакетами (Batch).
4. **Яркость**:
   * Перемещайте ползунок в секции **«Яркость»** для изменения интенсивности свечения в реальном времени.
5. **Экспорт**:
   * Нажмите **«Сгенерировать Arduino код»**, чтобы сохранить статическую картинку в формате скетча `.ino`. Полученный файл можно прошить в плату, и матрица будет светиться вашим рисунком автономно без подключения к компьютеру.

## 📡 Протокол Serial-команд (для разработчиков)

Программа общается с платой по интерфейсу UART (115200 baud). Вы можете отправлять следующие текстовые команды (заканчиваются символом `\n`):

| Команда | Описание | Пример |
|---|---|---|
| `CLEAR` | Очистка матрицы (все диоды выключены) | `CLEAR` |
| `SHOW` | Принудительный вывод буфера на матрицу | `SHOW` |
| `BRIGHT:<яркость>` | Установка общей яркости (0–255) | `BRIGHT:120` |
| `P:<индекс>,<R>,<G>,<B>` | Установка цвета одного пикселя (от 0 до 255) | `P:12,255,0,0` (сделать 12-й пиксель красным) |
| `FILL:<R>,<G>,<B>` | Заливка всей матрицы одним цветом | `FILL:0,255,0` (зеленая заливка) |
| `BATCH:<индекс>,<R>,<G>,<B>;<индекс>...` | Пакетная отправка группы светодиодов (без автовызова `show()`) | `BATCH:0,255,0,0;1,0,255,0` |

Каждая успешно выполненная команда возвращает ответ `OK`, неверная команда возвращает ошибку вида `ERR:CMD` или `ERR:FMT`.
